# MC6809E / MC68B09E

The CoCo 3 ships with the MC68B09E — same instruction set as the MC6809E used in the CoCo 1/2, but rated for the doubled 1.78 MHz bus. We can run Ladybug at the fast clock because we never enter the BASIC cassette/serial/disk I/O ROM paths.

## Programming model

```
 15                     0
┌─────────────────────────┐
│ X    (16-bit index/ptr) │
│ Y    (16-bit index/ptr) │
│ U    (user stack ptr)   │
│ S    (hardware stack)   │
│ PC                      │
├────────────┬────────────┤
│  A (8-bit) │  B (8-bit) │   D = A:B
└────────────┴────────────┘
 7        0    7        0
┌─────────┐  ┌─────────────┐
│   DP    │  │ E F H I N Z V C │   CC bits
└─────────┘  └─────────────┘
```

- `A`/`B` 8-bit accumulators; `D` is the 16-bit concatenation A:B.
- `X`, `Y`, `U`, `S` are full 16-bit pointer/index registers — all four work with every indexed addressing form, so we effectively have four independent index regs.
- `S` is the hardware stack used by the interrupt machinery. `U` is reserved for user-stack discipline.
- `DP` provides the upper byte of every direct-page address. Setting `DP` lets a hot variable area access in 2-byte / 4-cycle DP-relative form instead of 3-byte / 5-cycle extended.
- CC: `E` = entire-state stacked, `F` = FIRQ mask, `H` = half-carry, `I` = IRQ mask, `N` = negative, `Z` = zero, `V` = signed overflow, `C` = carry/borrow.

## Addressing modes

| Mode | Syntax | Notes |
|-|-|-|
| Inherent | `CLRA` | Implied; one byte. |
| Immediate | `LDA #$nn`, `LDX #$nnnn` | Operand follows op-code. |
| Extended | `STA $nnnn` | 16-bit absolute address. |
| Direct (DP) | `STA <$nn` | 8-bit operand combined with DP. |
| Extended Indirect | `STA [$nnnn]` | Effective address fetched from `$nnnn`. |
| Indexed (zero offset) | `LDA ,X` | |
| Indexed (5/8/16-bit offset) | `LDA n,X` | Constant signed offset. |
| Accumulator-offset indexed | `LDA B,X`, `LDA D,Y` | Runtime offset. |
| Auto-inc/dec | `LDA ,X+`, `,X++`, `,-X`, `,--X` | Mirrors push/pull. |
| Indexed indirect | `LDA [n,X]` | Any non-±1 indexed form, bracketed. |
| Relative | `BCS label`, `LBCS label` | All branches PC-relative; ±127 short, ±32 K long. |
| PC-relative | `LDA TABLE,PCR` | Position-independent data access. |

Indexed postbyte tables and TFR/EXG/PUSH/PULL register-code postbytes are catalogued in [../sources/coco-asm-tepolt.md](../sources/coco-asm-tepolt.md). For Ladybug, prefer DP-relative for hot per-frame state and PC-relative within position-independent code blocks (a cartridge ROM is naturally position-independent if we discipline ourselves).

## Interrupt sequences

| Type | Vector | Mask | Stacked regs | Sets/clears |
|-|-|-|-|-|
| RESET | `$FFFE/$FFFF` | — | none | clears DP, sets I and F |
| NMI   | `$FFFC/$FFFD` | unmaskable (after first S load) | all (PC,U,Y,X,DP,B,A,CC) | sets E, I, F |
| SWI   | `$FFFA/$FFFB` | unmaskable | all | sets E, I, F |
| IRQ   | `$FFF8/$FFF9` | I bit | all | sets E, I |
| FIRQ  | `$FFF6/$FFF7` | F bit | PC, CC only | clears E, sets I and F |
| SWI2  | `$FFF4/$FFF5` | unmaskable | all | sets E |
| SWI3  | `$FFF2/$FFF3` | unmaskable | all | sets E |

`RTI` inspects the popped CC: if E was set, pops everything; if E was clear (FIRQ path), pops only PC. **Implication:** an IRQ handler that wants to share state with FIRQ must do its own pushes — FIRQ does not save A/B/X/etc.

In ROM the vectors at `$FFF2-FFFF` are mirrored to RAM long-branch trampolines at `$FEEE-FEFF` (primary jump table) and to legacy 6-byte JMPs at `$0100-$010F` (secondary jump table). Both tables are writable, so a cartridge program installs its handlers there. See [../sources/coco3-asm-tepolt.md](../sources/coco3-asm-tepolt.md) Tables 6-1 / 6-2 / 6-3.

## Clock rate

| `$FFD8` | `$FFD9` | Speed mode |
|-|-|-|
| write to | — | 0.89 MHz (compat with cassette/serial/disk ROM I/O) |
| — | write to | 1.78 MHz (full speed; safe on a real CoCo 3 since the chip is the MC68B09E) |

Writing anything to either dedicated address flips the SAM R0 bit. We will set 1.78 MHz in our reset routine. (R1 also exists from the CoCo 1/2 era but has no separate effect on a CoCo 3.) See [../sources/coco-asm-tepolt.md](../sources/coco-asm-tepolt.md) Chapter 9 for the original SAM clock-rate table.
