# MC6809E / MC68B09E

The CoCo 3 ships with the MC68B09E — same instruction set as the MC6809E used in the CoCo 1/2, but rated for the doubled 1.78 MHz bus. We can run Ladybug at the fast clock because we never enter the BASIC cassette/serial/disk I/O ROM paths.

## Programming model

```
 15                     0
┌─────────────────────────┐
│ X    (16-bit index/ptr) │
│ Y    (16-bit index/ptr) │
│ U    (user stack ptr)   │
│ S    (hardware stack)   │
│ PC                      │
├────────────┬────────────┤
│  A (8-bit) │  B (8-bit) │   D = A:B
└────────────┴────────────┘
 7        0    7        0
┌─────────┐  ┌─────────────┐
│   DP    │  │ E F H I N Z V C │   CC bits
└─────────┘  └─────────────┘
```

- `A`/`B` 8-bit accumulators; `D` is the 16-bit concatenation A:B.
- `X`, `Y`, `U`, `S` are full 16-bit pointer/index registers — all four work with every indexed addressing form, so we effectively have four independent index regs.
- `S` is the hardware stack used by the interrupt machinery. `U` is reserved for user-stack discipline.
- `DP` provides the upper byte of every direct-page address. Setting `DP` lets a hot variable area access in 2-byte / 4-cycle DP-relative form instead of 3-byte / 5-cycle extended.
- CC: `E` = entire-state stacked, `F` = FIRQ mask, `H` = half-carry, `I` = IRQ mask, `N` = negative, `Z` = zero, `V` = signed overflow, `C` = carry/borrow.

## Addressing modes

| Mode | Syntax | Notes |
|-|-|-|
| Inherent | `CLRA` | Implied; one byte. |
| Immediate | `LDA #$nn`, `LDX #$nnnn` | Operand follows op-code. |
| Extended | `STA $nnnn` | 16-bit absolute address. |
| Direct (DP) | `STA <$nn` | 8-bit operand combined with DP. |
| Extended Indirect | `STA [$nnnn]` | Effective address fetched from `$nnnn`. |
| Indexed (zero offset) | `LDA ,X` | |
| Indexed (5/8/16-bit offset) | `LDA n,X` | Constant signed offset. |
| Accumulator-offset indexed | `LDA B,X`, `LDA D,Y` | Runtime offset. |
| Auto-inc/dec | `LDA ,X+`, `,X++`, `,-X`, `,--X` | Mirrors push/pull. |
| Indexed indirect | `LDA [n,X]` | Any non-±1 indexed form, bracketed. |
| Relative | `BCS label`, `LBCS label` | All branches PC-relative; ±127 short, ±32 K long. |
| PC-relative | `LDA TABLE,PCR` | Position-independent data access. |

Indexed postbyte tables and TFR/EXG/PUSH/PULL register-code postbytes are catalogued in [../sources/coco-asm-tepolt.md](../sources/coco-asm-tepolt.md). For Ladybug, prefer DP-relative for hot per-frame state and PC-relative within position-independent code blocks (a cartridge ROM is naturally position-independent if we discipline ourselves).

## Interrupt sequences

| Type | Vector | Mask | Stacked regs | Sets/clears |
|-|-|-|-|-|
| RESET | `$FFFE/$FFFF` | — | none | clears DP, sets I and F |
| NMI   | `$FFFC/$FFFD` | unmaskable (after first S load) | all (PC,U,Y,X,DP,B,A,CC) | sets E, I, F |
| SWI   | `$FFFA/$FFFB` | unmaskable | all | sets E, I, F |
| IRQ   | `$FFF8/$FFF9` | I bit | all | sets E, I |
| FIRQ  | `$FFF6/$FFF7` | F bit | PC, CC only | clears E, sets I and F |
| SWI2  | `$FFF4/$FFF5` | unmaskable | all | sets E |
| SWI3  | `$FFF2/$FFF3` | unmaskable | all | sets E |

`RTI` inspects the popped CC: if E was set, pops everything; if E was clear (FIRQ path), pops only PC. **Implication:** an IRQ handler that wants to share state with FIRQ must do its own pushes — FIRQ does not save A/B/X/etc.

In ROM the vectors at `$FFF2-FFFF` are mirrored to RAM long-branch trampolines at `$FEEE-FEFF` (primary jump table) and to legacy 6-byte JMPs at `$0100-$010F` (secondary jump table). Both tables are writable, so a cartridge program installs its handlers there. See [../sources/coco3-asm-tepolt.md](../sources/coco3-asm-tepolt.md) Tables 6-1 / 6-2 / 6-3.

## Clock rate

| `$FFD8` | `$FFD9` | Speed mode |
|-|-|-|
| write to | — | 0.89 MHz (compat with cassette/serial/disk ROM I/O) |
| — | write to | 1.78 MHz (full speed; safe on a real CoCo 3 since the chip is the MC68B09E) |

Writing anything to either dedicated address flips the SAM R0 bit. We will set 1.78 MHz in our reset routine. (R1 also exists from the CoCo 1/2 era but has no separate effect on a CoCo 3.) See [../sources/coco-asm-tepolt.md](../sources/coco-asm-tepolt.md) Chapter 9 for the original SAM clock-rate table.

## Mnemonic reference

Anchor targets for deep links from the `web/` retro-dev app's instruction-annotation panel. Each subsection summarises the family briefly; cycle-accurate details + per-encoding semantics live in the curated [opcode JSON](../../web/data/6809-opcodes.json).

### Load (LDA / LDB / LDD / LDX / LDY / LDS / LDU) {#load-ops}

Loads a register from memory or an immediate. 8-bit forms set `N` from bit 7 of the result, `Z` if the result is zero, and clear `V`. 16-bit forms set `N` from bit 15 instead. `C` is unaffected. Indexed and extended variants exist for every register.

### Store (STA / STB / STD / STX / STY / STS / STU) {#store-ops}

Stores a register to memory. CC effects mirror the load forms — `N` from the high bit of the value being stored, `Z` from zero-ness, `V` cleared. No immediate variant (you can't store to a constant).

### Effective-address load (LEAX / LEAY / LEAS / LEAU) {#lea-ops}

Compute an indexed effective address and load it into the destination register *without performing a memory access*. The common idiom for pointer arithmetic. `LEAX` and `LEAY` set `Z` (zero-result detection — useful for `LEAX -1,X` loop bottoms); `LEAS` and `LEAU` do NOT affect CC.

### Branch (BRA / BRN / BHI / BLS / BHS / BCC / BLO / BCS / BNE / BEQ / BVC / BVS / BPL / BMI / BGE / BLT / BGT / BLE / LBRA / LBSR / BSR) {#branch-ops}

PC-relative jumps. Short forms take an 8-bit signed offset (±127); long forms (page-2 prefixed) take a 16-bit signed offset (±32 K). BSR/LBSR additionally push the return PC to S. Branch *predicates* are CC-based — see the per-encoding `summary` in the opcode JSON for the exact predicate.

### Push / pull (PSHS / PSHU / PULS / PULU) {#push-pull-ops}

The postbyte selects which registers to transfer, with bit positions (high→low): PC, U, Y, X, DP, B, A, CC. Cycle cost is 5 + 1 per byte transferred. PULS/PULU restore CC if it's in the postbyte mask.

### Transfer / exchange (TFR / EXG) {#tfr-exg-ops}

TFR copies source→dest; EXG swaps. Postbyte holds two register codes (high nybble + low nybble). Codes: 0=D, 1=X, 2=Y, 3=U, 4=S, 5=PC, 8=A, 9=B, A=CC, B=DP. Mixing 8-bit and 16-bit codes is allowed; the manual specifies the conversion.

### Return / halt (RTS / RTI / SWI / SYNC / NOP / CWAI) {#return-halt-ops}

`RTS` pops PC. `RTI` pops a stack frame whose size depends on the E bit in the stacked CC (FIRQ leaves 6 cycles, IRQ/SWI/NMI 15). `SWI` is a software trap vectoring through `$FFFA-FFFB` — note the [XRoar 1.10 segfault](../tooling/xroar.md) when `SWI` is at a `-trap` point. `SYNC` halts the CPU until an interrupt. `NOP` is a 2-cycle no-op. `CWAI` ANDs an immediate into CC then halts until interrupt.

### CC manipulation (ORCC / ANDCC) {#cc-ops}

Bit-wise OR / AND an 8-bit immediate into CC. Used to set or clear mask bits — e.g. `ORCC #$50` to mask both IRQ and FIRQ during a critical section.

### Arithmetic-and-bit (CLR / CLRA / CLRB / INC / INCA / INCB / DEC / DECA / DECB / TST / NEG / COM / ASL / ASR / LSR / ROL / ROR / ADD / SUB / CMP / AND / OR / EOR / BIT / MUL / SEX / ABX / DAA / ADC / SBC) {#arith-bit-ops}

Standard 6809 arithmetic, logical, shift, and bit-test instructions. CC effects follow the conventional rules: `N` from result high bit, `Z` from zero-ness, `V` for signed overflow, `C` for unsigned overflow (add) or borrow (subtract / shift-out). See the per-encoding `cc` field in the opcode JSON for the precise effect of each mnemonic.
